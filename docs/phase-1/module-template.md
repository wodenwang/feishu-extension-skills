# 模块文档模板

所有第一阶段模块文档尽量按以下结构书写：

## 1. 模块定位

- 模块职责
- 为什么属于第一阶段

## 2. 范围

- In scope
- Out of scope

## 3. 依赖关系

- 上游依赖
- 下游调用方

## 4. 代码结构规划

- 目录
- 文件职责

## 5. 核心数据模型

- 输入模型
- 输出模型
- 错误模型

## 6. 对外动作

- action 列表
- CLI 调用方式

## 7. 飞书 API / 外部接口

- 主要接口
- 请求方式

## 8. 权限与配置

- 飞书权限
- 调用时凭证
- 环境变量兜底

## 9. 测试与验收

- 单元测试
- 冒烟测试
- 验收标准

## 10. 后续扩展

- 第二阶段或第三阶段如何继续扩
